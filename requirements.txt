Create a neovim plugin for capturing daily notes.  

The plugin should provide the ability to do the following:
1. Have a keybinding that creates or opens a daily note file named with the current date (e.g., "2023-03-15.md") in a specified directory. 
2. Automatically insert a header with the current date and a predefined template into the daily note file if it is newly created.
3. Allow customization of the notes directory and the template through plugin settings.
4. Provide a command to list all daily notes and open a selected one.
5. Ensure compatibility with popular Neovim plugins like Telescope for enhanced note searching and navigation.
6. Include error handling for file operations and user feedback for actions performed.
7. Write clear documentation on how to install, configure, and use the plugin.
8. Ensure the plugin is efficient and does not introduce noticeable lag in Neovim's performance.
9. Follow best practices for Neovim plugin development, including using Lua for implementation.
10. Include unit tests for core functionalities of the plugin.
11. Provide a way to back up daily notes to a specified location or cloud service.
12. Allow tagging of notes and provide a way to filter notes by tags.
13. Support for markdown formatting and preview within Neovim.
14. Implement a search functionality within daily notes to find specific content quickly.
15. Ensure the plugin is compatible with both Linux and Windows operating systems.
16. Regularly update the plugin to fix bugs, add new features, and ensure compatibility with
    the latest version of Neovim.
17. Set up a GitHub repository for the plugin to facilitate collaboration and issue tracking.
18. keybinding to quickly toggle between daily notes and a general notes file.
19. Provide a way to archive old daily notes automatically after a certain period.
20. customizable date formats for daily note filenames and headers.
21. Add frontmatter support for daily notes to include metadata like tags, created date, and modified date.
22. Should have a user function that can generate a report of all uncompleted tasks from daily notes.
23. All note types should support linking and backlinking between notes.
24. Set the archive feature to 0 days by default but allow users to customize this setting.
25. general notes should be broken into multiple files using the PARA (Project, Area, Resource, Archive) method. The general.md file should be a dashboard that links to these files.
